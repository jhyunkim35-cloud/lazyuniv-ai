DATA MODEL (Firestore + IDB mirror):
- users/{uid}/timetable/{entryId}
- Fields:
  - entryId: stable id (use crypto.randomUUID() or similar)
  - courseName: string (e.g. "산업심리학")
  - folderId: string | null (optional link to existing folder so timetable entry can map to that folder's notes)
  - dayOfWeek: 0-6 (0=Sun, 6=Sat)
  - startTime: 'HH:MM'
  - endTime: 'HH:MM'
  - autoRecord: boolean (default false; future: scheduler hook)
  - createdAt, updatedAt: ISO strings

NEW FILE /public/js/timetable.js exposing window.* functions:
- saveTimetableEntry(entry) -- writes to FS users/{uid}/timetable/{entryId} with merge:true; mirrors to IDB. Call userTimetableRef pattern matching existing firestore_sync helpers (see firestore_sync.js for the FS path / db ref pattern).
- getTimetableEntry(entryId) -- reads one entry, FS first with IDB fallback.
- listTimetableEntries() -- returns array of all entries for current user, sorted by dayOfWeek ASC then startTime ASC.
- deleteTimetableEntry(entryId) -- deletes from FS and IDB.
- newTimetableEntry(partial) -- creates a fresh entry with defaults: dayOfWeek=1 (Mon), startTime='09:00', endTime='10:30', autoRecord=false. Spreads partial on top. Returns entry object (NOT saved yet).
