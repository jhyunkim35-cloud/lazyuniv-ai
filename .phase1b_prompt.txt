NEW FILE /public/js/srs.js exposing window.* functions:
1. computeNextSrs(card, quality) -- pure function, returns updated card object
   SM-2 algorithm:
   - quality is 0-5
   - if quality < 3:
     - card.interval = 1
     - card.repetitions = 0
   - else:
     - if card.repetitions == 0: interval = 1
     - else if card.repetitions == 1: interval = 6
     - else: interval = round(card.interval * card.easeFactor)
     - card.repetitions += 1
   - easeFactor = max(1.3, easeFactor + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02)))
   - card.nextReviewDate = today + interval days, formatted YYYY-MM-DD
   - card.lastReviewDate = today YYYY-MM-DD
   - card.lastQuality = quality
   - return card

2. cardIdFor(folderId, noteId, sectionTitle) -- deterministic id: simple hash like sha-like or just slug + base64; doesn't matter as long as stable

3. saveSrsCard(card) -- writes to Firestore at users/{uid}/srsCards/{cardId} via existing db ref pattern (look at firestore_sync.js for patterns), AND mirrors to IDB for offline. Use { merge: true }.

4. getSrsCard(cardId) -- read one card; returns null if not found. Try Firestore first, fall back to IDB.

5. getDueCards(folderId, todayYmd) -- returns array of cards in folderId with nextReviewDate <= todayYmd, sorted ascending by nextReviewDate. Limit by folder's effectiveDailyTarget (call existing window.effectiveDailyTarget(plan, notesCount)).

6. gradeCard(cardId, quality) -- convenience: load card (or create with defaults if missing: interval=0, repetitions=0, easeFactor=2.5), compute next, save, return updated.
